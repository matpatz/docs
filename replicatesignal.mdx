title: "replicatesignal"

```luau
replicatesignal(signal: RBXScriptSignal, ...: any): ()
```

Pass `remote.OnServerEvent{:luau}` with `LocalPlayer` first, then your args. The server receives `(LocalPlayer, ...args){:luau}` - same shape as a normal `remote:FireServer(...args){:luau}` call.

Local-only signals don't fire through this - reach for `firesignal{:luau}` instead.

Most commonly used for server-side actions your own script can't normally trigger - firing a hooked `RemoteEvent{:luau}`, or forcing a respawn without dying. <a href="/docs/contributors#lishbon" title="Contributed by lishbon"><img src={pfpLishbon.src} alt="lishbon" style="width:1.25rem;height:1.25rem;border-radius:9999px;display:inline-block;vertical-align:middle;object-fit:cover;margin-left:0.25rem;" /></a>

## Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `signal` | `RBXScriptSignal` | The signal to replicate. |
| `...` | `any` | Args matching the signal's expected signature. |

## Returns

<code title="the function does not return a value.">void</code>
