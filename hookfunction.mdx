title: "hookfunction"
---

```luau
hookfunction(target: function, hook: function): function
```

Once hooked, every call to `target{:luau}` runs `hook{:luau}` instead.

Most commonly used to intercept a function (i.e. `print{:luau}`), or wrap its calls without losing access to the original.

Closure types may differ between `target{:luau}` and `hook{:luau}` - Luau-to-Luau, C-to-C, or any cross-direction.

`newcclosure{:luau}`-wrapped targets work, and the hook can carry a different upvalue count from the target.

## Parameters

Both arguments are required, otherwise - errors.

| Parameter | Type | Description |
|-----------|------|-------------|
| `target`  | `function` | The function to replace. Lclosure, cclosure, or `newcclosure{:luau}`-wrapped - all accepted. |
| `hook`    | `function` | The function that runs in `target{:luau}`'s place. |

## Returns

`function` - original `target{:luau}` function, **before** the hook was applied. Use it from inside `hook{:luau}` to trigger pre-hook behavior - `return original(...){:luau}`.

Handles don't go stale on re-hook. Hook `target{:luau}` a second time and the earlier handle still calls the body that was live when you first got it back.

## Aliases

- `hookfunc{:luau}`
- `replaceclosure{:luau}`

## Example

<CodeGroup>

```luau Example 1
hookfunction(target: function, hook: function): function
```

```luau Example 2
local Old
Old = hookfunction(StockUpdated, function(tbl, x, ...)
    Events.StockUpdate(tbl)

    return Old(tbl, x, ...)
end)
```

</CodeGroup>

<Tip>
  Always call the **returned handle** from inside your hook, not the target's original name. Calling `StockUpdated(...)` inside the hook re-enters the hook and recurses until the script crashes. `Old(...)` skips the hook and calls the pre-hook body.
</Tip>
