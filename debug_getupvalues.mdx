```luau
debug.getupvalues(func: function | number): {any}
```

New to upvalues? [Learn more here](/docs/notes/#upvalues).

<Aside type="note">
Give it a number instead of a function and it reads a stack level, exactly like `debug.getupvalue{:luau}`.
</Aside>

Most commonly used to snapshot a closure's captured state in one call: find an unknown
function with `getgc{:luau}`, dump its upvalues, and read off the objects it is holding
before touching it.

## Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `func` | <span style="white-space: nowrap;">`function` or `number`</span> | The closure to read. |

## Returns

`{any}` - an array-style table of the captured upvalues, in the order they were
captured. A closure with no upvalues returns an empty table, never `nil{:luau}`.

## Aliases

- `getupvalues{:luau}`

## Example

<CodeGroup>

```luau Example 1
debug.getupvalues(func: function | number): {any}
```

```luau Example 2
local config = {walkspeed = 16}
local active = true

local function fn()
    active = not active -- assigned, so 'active' stays a real upvalue
    config.walkspeed += 1
end

for i, v in pairs(debug.getupvalues(fn)) do
    print(`upvalue {i}: {typeof(v)}`)
end
```
